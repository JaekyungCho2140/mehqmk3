# memoQ Project Home Documentation

## Overview
When you create or open a local project, it will appear in a tab called Project Home. From this interface, users can access documents, resources, and project settings without opening individual files.

## Primary Functions
The Project Home tab serves as a central hub where users can:
- View and manage project documents
- Access translation resources
- Modify project configuration
- Generate project reports

## Navigation Access

Users can reach Project Home through several methods:

**Opening Local Projects:**
On the Dashboard, find it in the list, and double-click its name. Or, right-click the project name, and choose Open.

**Creating New Projects:**
Navigate to the Project ribbon and select New Project, which launches the template-based project creation window.

**Checking Out Online Projects:**
Use the "Check Out from Server" option on the Project ribbon to create local copies of online projects from memoQ TMS.

## Core Panes

| Pane | Purpose |
|------|---------|
| Translations | Document management and translation workflow |
| LiveDocs | Access earlier translation suggestions |
| Translation Memories | Manage translation memory resources |
| Term Bases | Manage terminology databases |
| Muses | Additional suggestion resources |
| Settings | Configure segmentation rules and QA settings |
| Overview | Generate project reports |
| People | Assign work (project manager only) |

## Closing Projects

Users can close projects by clicking the close icon in the Project Home tab or selecting "Close Project" from the Project ribbon.
